# 채널스코프 — 유튜브 채널 탐색기

유튜브 채널을 등급별(급상승/메가/대형/중형/소형)로 탐색하고,
채널별 **구독자수 · 총 조회수 · 개설일 · 인기 영상 · 댓글**을 확인하는 사이트입니다.

**API 키 없이 동작합니다.** 유튜브 웹사이트가 내부적으로 쓰는 공개 엔드포인트(Innertube)를
직접 호출하는 방식이라 키 발급·로그인이 전혀 필요 없습니다.

## 실행 방법

### 가장 쉬운 방법 (더블클릭)

폴더 안의 **`채널스코프 실행.command`** 파일을 더블클릭하면 서버가 켜지고
브라우저가 자동으로 열립니다. (창을 닫으면 서버도 함께 종료)

> ⚠️ **주의:** `public/index.html` 파일을 직접 더블클릭해서 열면 **빈 화면**이 됩니다.
> 이 앱은 화면을 그릴 때 서버에서 실시간 유튜브 데이터를 받아오므로, 반드시 서버를
> 켠 상태에서 http://localhost:3456 으로 접속해야 합니다.

### 터미널로 실행

```bash
cd "유튜브 채널 탐색기"
node server.js
```

브라우저에서 http://localhost:3456 접속. (별도 설치 필요 없음 — Node 18+ 만 있으면 됩니다)

- 기본 모드: **실시간 데이터** (우측 상단 🟢 표시)
- 데모 데이터로 실행: `DATA_MODE=mock node server.js`
- 유튜브 요청이 실패하면 자동으로 데모 데이터로 대체됩니다 (🟠 표시)

## 채널 풀 자동 수집

서버가 시작되면 48개 카테고리 검색어로 **200개 이상의 채널을 자동 수집**해
`data/pool.json`에 저장합니다. 이어서 백그라운드로 각 채널의 개설일·총조회수·배너를
채워 넣습니다 (수집 중에는 화면 상단에 진행률이 표시되고 자동 갱신됩니다).

- 수집된 풀은 24시간 동안 재사용됩니다. 강제 재수집: `REFRESH_POOL=1 node server.js`
- 카테고리(검색어)를 추가하려면 `server.js`의 `CATEGORIES` 배열에 넣고 재시작하면
  새 검색어만 증분 수집됩니다.
- 카테고리당 수집 채널 수는 `PER_CATEGORY`(기본 4)로 조절합니다.

## 사용법

- **메인 화면**: 수집된 채널 풀 전체가 컴팩트 카드로 표시됩니다.
- **검색**: 검색창에 입력하면 실제 유튜브에서 채널을 검색합니다.
- **등급 탭**: 급상승/메가/대형/중형/소형으로 필터링. 카테고리 드롭다운과 조합 가능.
- **채널 클릭**: 구독자수, 총 조회수, 개설일, 인기 영상 TOP(클릭하면 유튜브로 이동), 인기 댓글 표시.

## 데이터 내보내기

수집한 채널 전체를 파일로 내려받을 수 있습니다 (필터 옆 버튼):

- **📊 엑셀** (`/api/export.xlsx`) — 항목별로 정리된 진짜 엑셀 파일. 3개 시트로 구성:
  - `채널목록`: 카테고리 · 채널명 · 핸들 · 등급 · 급상승 · 구독자수 · 총조회수 · 영상수 · 개설일 · 국가 · URL (카테고리→구독자순 정렬, 헤더 고정)
  - `카테고리별요약`: 카테고리마다 채널수 · 등급 분포 · 구독자/조회수 합계
  - `등급별요약`: 메가/대형/중형/소형/급상승별 채널수와 합계
  - 외부 라이브러리 없이 순수 Node(`xlsx-writer.js`)로 생성합니다.
- **📥 CSV** (`/api/export.csv`) — 엑셀에서 바로 열리는 CSV (한글 깨짐 방지 BOM 포함)
- **📥 JSON** (`/api/export.json`) — 전체 채널 데이터 JSON

> 오프라인 대시보드(`채널스코프.html`)와 별개로, 엑셀 파일도 `node build-standalone.js`가 아니라
> 서버 실행 중 위 버튼으로 언제든 최신 데이터로 받을 수 있습니다.

## 채널 등급 기준

| 등급 | 기준 |
|---|---|
| 💎 메가 | 구독자 500만 이상 |
| 🏆 대형 | 구독자 100만 이상 |
| ⭐ 중형 | 구독자 10만 이상 |
| 🌱 소형 | 구독자 10만 미만 |
| 🚀 급상승 | 실시간: 개설 3년 미만 + 구독자 5만 이상 / 데모: 30일 성장률 8% 이상 |

기준값은 `server.js` 상단의 `TIERS`, `RISING_GROWTH`, `RISING_MAX_AGE_YEARS`, `RISING_MIN_SUBS`에서 수정할 수 있습니다.

## 알아두면 좋은 점

- 유튜브에 부담을 주지 않도록 모든 요청은 **10분간 캐시**됩니다.
- 유튜브가 UI를 개편하면 응답 구조가 바뀌어 일부 데이터가 안 나올 수 있습니다.
  그런 경우에도 사이트는 동작하며, 해당 항목만 `-`로 표시됩니다.
- 실시간 모드의 구독자수는 유튜브가 공개하는 근사값("318만명")입니다.
- 유튜브는 "최근 30일 성장률"을 공개하지 않으므로, 실시간 모드의 급상승 판정은
  채널 나이+규모로 근사합니다. 정확한 성장률이 필요하면 주기적으로 수치를 저장해
  비교하는 기능을 추가할 수 있습니다.

## 구조

```
유튜브 채널 탐색기/
├── server.js              # HTTP 서버 + 채널 풀 수집 + 등급 분류 + 캐시 (의존성 없음)
├── scraper.js             # 키 없는 유튜브 데이터 수집 모듈 (Innertube)
├── data/
│   ├── pool.json          # 자동 수집된 채널 풀 (서버가 생성/갱신)
│   └── mock-channels.json # 데모 채널 데이터 (20개 채널)
└── public/
    ├── index.html
    ├── style.css
    └── app.js             # 프론트엔드 (해시 라우팅 SPA)
```
